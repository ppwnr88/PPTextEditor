# Code of Conduct

## Our Standard

We want PPText Editor to be a project where people can ask questions, share ideas, and contribute improvements without being dismissed or disrespected.

Examples of behavior that helps create a positive environment:

- being respectful and patient
- giving constructive feedback
- assuming good intent
- welcoming different experience levels
- focusing on the problem, not the person

Examples of unacceptable behavior:

- harassment or personal attacks
- insulting or demeaning language
- hostile or exclusionary behavior
- publishing private information without permission
- repeated bad-faith disruption of discussions

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- code review discussions
- project-related communication spaces

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that do not align with this Code of Conduct.

## Reporting

If you experience or witness unacceptable behavior, report it through a GitHub issue only if it is appropriate to do so publicly.
If a private channel is needed later, project maintainers can define one.

## Notes

This project is small and practical. The goal of this document is simple: keep collaboration respectful, useful, and safe.
